# ハドフ巡りマップ 仕様書

## 概要

HARD OFFグループ（ハードオフ／オフハウス／ホビーオフ／モードオフ／ガレージオフ／リカーオフ）の全国の店舗を地図上に表示し、訪問済みの店舗をチェックインして記録できるWebアプリ。

- 対象範囲: 全国47都道府県、996店舗
- 対象ブランド: ハードオフ・オフハウス・ホビーオフ・モードオフ・ガレージオフ・リカーオフの6ブランド（ブックオフは対象外）
- 単一ページ構成（`public/index.html`）。地図と店舗一覧は別ページではなく、一覧は地図の下からスライドして開くシートとして実装している

## 技術構成

| 項目 | 内容 |
|---|---|
| サーバー | Node.js + Express（`server.js`）。`public/`を静的配信するのみ |
| フロントエンド | TypeScript（`src/*.ts`）。`npm run build`で`tsc`が`public/js/*.js`にコンパイルし、`public/index.html`が`<script src>`で読み込む。`npm start`/`npm run dev`は起動前に自動でビルドする |
| 地図 | Leaflet 1.9.4 + OpenStreetMapタイル（型定義は`@types/leaflet`） |
| データ永続化 | チェックイン状態は `localStorage`（ブラウザ標準の永続ストレージ、同一オリジンならページを閉じても消えない）に保存。使えない環境ではその旨をトーストで通知し、画面を開いている間だけメモリ上で有効 |
| 店舗データ | `data/stores.json`（`scripts/fetch-stores.js` でhardoff.co.jpから取得・生成）を元に `public/js/stores.js` 内の `STORES` 定数へ直接埋め込み |

## 画面構成

### ヘッダーパネル（左上固定）

- アプリ名「ハドフ巡りマップ」とサブタイトル「JAPAN β / 全国 {店舗数}店舗」
- 「めぐり達成」進捗タグ: チェックイン済み数 / 全店舗数（全国・全ブランド基準、フィルタの影響を受けない）
- 進捗バー（達成率に応じて伸びる）
- 都道府県セレクトボックス（下記「都道府県フィルタ」参照）

### 地図

- 初期表示: 日本全体が収まるズーム・中心位置
- ピンをタップするとポップアップで店舗名・住所・公式ページ／Googleマップへのリンク・チェックインボタン・ナビリンク（Google/Apple/Yahoo!）を表示

### フィルタチップ（下部、横スクロール）

- 「すべて」＋ブランドごとのチップ（ハードオフ／オフハウス／ホビーオフ／モードオフ／ガレージオフ／リカーオフ）
- 「未訪問のみ」トグル
- ブランドフィルタ・未訪問フィルタは**地図上のピン表示のみ**に影響し、下記の店舗一覧シートには影響しない

### 都道府県フィルタ

- ヘッダー内のセレクトボックスで都道府県を選択すると、**地図のピンと店舗一覧シートの両方**が選択した都道府県のみに絞り込まれる
- 「全都道府県」を選ぶと全国表示に戻る
- 選択中の都道府県名と、絞り込み後の店舗数がヘッダーのサブタイトルに反映される

### 店舗一覧シート（右下の≡ボタンから下にスライドして開く）

別ページではなく、地図画面の下から出てくるモーダルシートとして実装。閉じるとまた地図が全画面表示に戻る。

- タブ:「すべて」「未訪問」「チェック済み」（都道府県フィルタと併用可能）
- 上部に都道府県ジャンプ用のチップ列（横スクロール、クリックで該当セクションへ自動展開＋スクロール）
- `PREF_ORDER`（北海道→沖縄）順に都道府県ごとへセクション分けして表示。**各セクションはアコーディオンで初期状態は全て折りたたみ**（996店舗を一度に表示すると長すぎるため）。見出しをタップすると開閉する。都道府県セレクトで1県のみに絞り込んでいる場合はそのセクションを常に展開表示する
- 都道府県見出しの横に**その県の全店舗を一括チェックイン/解除するチェックボックス**がある。県内が全てチェック済みなら✓、一部だけなら半チェック（indeterminate）表示になる
- 各店舗カードの内容:
  - チェックボックス（チェックイン状態のトグル。カード全体がタップ領域）
  - ブランドバッジ（`BRANDS`の色・ラベルを使用）
  - 店舗名・郵便番号・住所
  - チェックイン済みの場合はチェックイン日を表示し、カードの背景・枠線が緑になる
  - リンク行: Googleマップ（経路検索）／Appleマップ／Yahoo!カーナビ／「地図で見る」（シートを閉じて地図上の該当ピンのポップアップを開く）／口コミ(Google)（Googleマップの検索結果ページへのリンク。店舗のGoogle Place IDを保持していないため、店名+住所での検索リンクを代用している）
- チェックボックス操作（個別・都道府県一括とも）は地図上のピンの見た目・進捗タグにも即時反映される（同じページ内の同じ`checkins`オブジェクトを操作しているため、ページ間の同期は発生しない）

### 現在地ボタン

- タップで現在地を取得し、地図を現在地へ移動。以降は現在地追従モードになり、地図をドラッグすると解除される

## ピンのデザイン

### 未訪問ピン

背景色はブランドごとの実際のロゴカラーに合わせている。

| ブランド | 内部キー | マーク文字 | 背景色 | 文字色 |
|---|---|---|---|---|
| ハードオフ | `hoff` | H | `#183F92` | 白 |
| オフハウス | `offhouse` | O | `#317245` | 白 |
| ホビーオフ | `hobbyoff` | B | `#D22D26` | `#FDF150`（黄） |
| モードオフ | `modeoff` | M | `#D22D26` | 白 |
| ガレージオフ | `garageoff` | G | `#4B5563`（暫定色。ロゴ画像未確認のため） | 白 |
| リカーオフ | `liquoroff` | L | `#9A3244` | 白 |

### チェックイン済みピン

背景を白に反転し、枠線と文字をブランド色にする（ブランドが判別できるようにしつつ、訪問済みであることが一目でわかるデザイン）。

### 同一座標の複合店舗（同じ住所に複数ブランドが入居している店舗）

ピンが重ならないよう、経度方向にのみ交互（右→左→右…）にずらして表示する。

## データ仕様

`STORES` 配列の各要素:

```json
{
  "id": "101024",
  "brand": "hoff",
  "name": "ハドフ新座店",
  "zip": "352-0011",
  "address": "埼玉県新座市野火止8-4-45",
  "prefecture": "埼玉県",
  "lat": 35.795471,
  "lng": 139.572445
}
```

- `lat`/`lng` は `data/stores.json` 由来（hardoff.co.jpの各店舗詳細ページに埋め込まれたGoogleマップリンクの座標）。町丁目レベルの粗い推定ではなく、公式サイトが保持する座標を使用している
- 表示用の位置（`dlat`/`dlng`）は上記の複合店舗オフセットを適用した後の値で、実行時に算出される

## チェックイン仕様

- 地図のポップアップ内ボタン、または一覧シートのチェックボックスでチェックイン／取り消しをトグル（同じ`setCheckin()`関数を共通利用）
- チェックイン時は当日の日付（`YYYY-MM-DD`）を記録
- 状態変更のたびに進捗タグ・進捗バー・ピンの見た目・ポップアップ内容・一覧シートを再描画する
- 状態は`localStorage`に保存され、ページを再読み込みしても保持される

## 既知の制約・今後の検討事項

- ブックオフは意図的に対象外（要望により除外）
- ガレージオフの背景色は実物ロゴを確認できていないため暫定値
- 全国996店舗のピンを常時DOM生成しているため、低ズーム時は密集地でピンが視覚的に重なる（マーカークラスタリングは未導入）
- 店舗一覧シートはブランドフィルタと連動していない（都道府県フィルタのみ連動）
- 「口コミ(Google)」リンクはGoogleのPlace IDを使った直接的な口コミタブへのリンクではなく、店名+住所での検索結果ページ
- チェックイン状態は`localStorage`（ブラウザ単位）に保存されるため、別のブラウザ・別の端末とは同期しない。複数端末で共有したい場合はサーバー側での永続化が別途必要
